# 财务对账逻辑说明

## 输入来源

当前工具使用两张表：

- 原始支付宝账单
- 财务分成明细表

两者都支持直接上传 `Excel` 或 `zip`。

## 已确认规则

### 1. 分类

优先从 `商品名称` 提取游戏名，例如：

- `梦幻小院30元游戏礼包` -> `梦幻小院`
- `逍遥仙648元游戏礼包` -> `逍遥仙`

使用模式：

```regex
^(.*?)(\d+(?:\.\d+)?)元(?:游戏)?礼包$
```

### 2. 收入

原始支付宝账单中，`业务描述` 以 `0010001|` 开头的 `收入（+元）` 按游戏汇总。

### 3. 支出

原始支付宝账单中，`业务描述` 以 `0070002|` 开头的 `支出（-元）` 按游戏汇总。

### 4. 手续费

原始支付宝账单中，手续费优先按以下规则识别：

- `业务描述` 以 `0030003|` 开头
- 如果 `业务描述` 为空，但 `备注` 中包含 `天猫佣金（类目系统）` 且能提取出订单号，也按手续费处理

公式为：

```text
手续费 = 支出 - 收入
```

### 5. 基础软件费

原始支付宝账单中，`业务描述` 以 `0030130|` 开头的 `支出（-元）` 按游戏汇总。

### 6. 分成

分成不再使用费率估算。

当前正式规则：

```text
分成 = 财务分成明细表中同游戏的 sum(扣费金额)
```

财务分成明细表主要使用字段：

- `商品名称`
- `扣费金额`
- `交易主订单号`

### 7. 推理意见

如果发现：

- 原始支付宝账单中有退款
- 财务分成明细表中同订单已有正分成
- 但缺少对应负分成冲回

工具会额外给出一份自动推理结果：

- 推理规则：`推理补回分成 = -1 * 同订单原始正分成`
- 推理结果不会覆盖原始 `对账汇总`
- 会单独输出到 `推理意见` sheet 供财务决定是否采纳

如果当前账期里无法直接按同订单找到正分成，工具还会继续参考：

- 当前分成明细表中 `同游戏 + 同金额档位` 的稳定分成
- `输出结果` 目录中历史工作簿 `推理意见` 里的订单级明细

### 8. 其他

无法按当前规则归类到具体游戏的流水，统一汇总到 `其他`。

需要特别区分：

- `其他收入`
- `其他支出`
- `其他手续费`
- `其他基础软件费`

### 9. 转出到网商银行

原始支付宝账单中，`备注` 包含 `转出到网商银行` 的流水，统一汇总到 `转出到网商银行`。

## 当前工具行为

工具会输出：

- 正常游戏分类
- `其他`
- `转出到网商银行`

如果检测到需要复核的风险项，还会额外输出：

- `异常检查`
  - 用于提示退款缺少负分成冲回
  - 孤立手续费
  - 孤立基础软件费
  - 分成未归类

如果存在可自动判断的退款缺少负分成冲回，还会额外输出：

- `推理意见`
  - 展示自动补回后的分成结果
  - 同时附带订单级推理明细

如果未提供财务分成明细表：

- 结果仍会生成
- `分成` 列保持空白
