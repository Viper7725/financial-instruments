# 本地游戏对账工具

## 当前版本说明

这个工具当前使用两类原始文件：

- 原始支付宝账单：支持 `Excel` 或 `zip`
- 财务分成明细表：支持 `Excel` 或 `zip`

其中：

- `分成` 以财务分成明细表中的 `扣费金额` 为准
- 不再使用费率估算，也不再提供人工修正文件入口
- 如果发现 `退款缺少负分成冲回`，工具会新增 `推理意见` sheet，给出自动补回后的参考结果
- 如果当前账期证据不足，工具会继续参考 `输出结果` 目录里过往结果中的 `推理意见` 明细
- `备注` 包含 `转出到网商银行` 的流水会单独汇总到 `转出到网商银行`
- 无法归类到具体游戏的流水会汇总到 `其他`

## 主要文件

- `小曼游戏对账.exe`
  - 推荐直接双击使用的无黑框启动器
- `Start-ReconcileTool.ps1`
  - 图形界面脚本
- `Reconcile-AlipayWorkbook.ps1`
  - 核心对账脚本
- `INFERENCE.md`
  - 当前对账逻辑说明

## 图形界面使用

1. 打开 `小曼游戏对账.exe`
2. 选择 `原始支付宝账单`
3. 确认 `对账结果保存位置`
4. 选择 `财务分成明细表`
5. 点击 `开始对账`

默认情况下，结果会保存到工具目录下的 `输出结果\YYYY-MM` 文件夹。

如果文件名里暂时识别不出月份，会先落到 `输出结果\未分类`。

默认文件名会尽量使用自然格式，例如：

- `对账结果-2025年08月.xlsx`

## 命令行使用

```powershell
powershell -ExecutionPolicy Bypass -File .\Reconcile-AlipayWorkbook.ps1 `
  -InputPath "G:\新建文件夹\账务组合查询25.8月.xls.zip" `
  -ShareStatementPath "G:\新建文件夹\南锦25.8月.xlsx" `
  -OutputPath ".\result.xlsx"
```

## 输出内容

输出文件包含以下列：

- `分类`
- `求和项:收入（+元）`
- `求和项:支出（-元）`
- `求和项:分成`
- `求和项:手续费`
- `求和项:基础软件费`

并且会额外保留：

- `其他`
- `转出到网商银行`

如果工具发现需要复核的情况，还会额外生成：

- `异常检查`
  - 例如退款缺少负分成冲回
  - 孤立手续费 / 基础软件费
  - 分成未归类

如果发现可自动推理的退款负分成缺失，还会额外生成：

- `推理意见`
  - 保留一份自动补回后的参考汇总
  - 下方附带订单级推理明细，便于财务核对

## 说明

- 脚本依赖本机已安装的 Excel
- 支持直接上传 `zip`，工具会自动解压
- 如果结果需要微调，可以直接修改导出的 Excel
- 大文件通常需要十几秒到几十秒
